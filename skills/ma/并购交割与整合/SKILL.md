## 轻量入口

本文件是触发入口。详细交割清单、整合框架已迁移至 `references/` 和 `checklists/` 目录。

## 何时使用

- 用户提到「交割」「closing」「过户」「工商变更」「整合」「交接」
- 交易文件签署后需要准备交割
- 交割后需要做整合管理
- 法律工作总控路由到本 Skill

## 执行顺序

### 交割阶段

1. 从交易文件中提取交割条件清单（Conditions Precedent）
2. 逐项核查交割条件满足情况
3. 准备交割文件包
4. 执行交割流程
5. 撰写交割备忘录
6. 出稿前审查

### 整合阶段

1. 评估整合风险（法律实体、劳动关系、合同、知识产权、合规体系）
2. 制定整合方案
3. 跟踪整合进度
4. 形成整合风险评估报告

## 交割条件核查清单

### 通用交割条件

- [ ] 交易各方内部决议/授权文件
- [ ] 反垄断审查（如触发申报标准）
- [ ] 行业监管审批（外商投资准入、金融牌照等）
- [ ] 国资审批/备案（涉及国有企业）
- [ ] 第三方同意（债权人、合作方、房东等）
- [ ] 无重大不利变化（MAC）确认
- [ ] 目标公司陈述保证在交割日仍真实
- [ ] 交割证明文件（certificate）

### 股权交易专项

- [ ] 工商变更登记（营业执照换发）
- [ ] 股东名册更新
- [ ] 章程修订
- [ ] 印章/证照/账簿交接清单
- [ ] 银行账户/网银U盾交接

### 资产交易专项

- [ ] 不动产权属变更登记
- [ ] 知识产权转让登记
- [ ] 车辆过户
- [ ] 设备清点和交付
- [ ] 存货盘点确认

## 整合风险维度

| 维度 | 主要风险 | 关注点 |
|:---|:---|:---|
| 法律实体 | 法人资格保留/注销 | 是否吸收合并、分公司整合 |
| 劳动关系 | 员工转移、裁员 | 劳动合同变更、经济补偿 |
| 合同关系 | 合同主体变更 | 是否需第三方同意 |
| 知识产权 | 权属转移、许可延续 | 商标专利变更登记 |
| 合规体系 | 监管牌照更名 | 行业许可变更 |
| IT与数据 | 数据迁移、系统合并 | 个人信息保护合规 |

## 输出底线

- 交割未完成前，不得将交割条件标注为已满足
- 整合方案不得替代企业管理层决策
- 工商变更流程必须核实当地具体要求
