1. 确认交易基础信息（尽调结论、交易双方需求、商业目标）
2. 选择交易类型：股权收购 vs 资产收购 vs 合并
3. 设计交易结构（直接/间接/SPV）
4. 税务筹划框架（企业所得税、增值税、印花税、土增税）
5. 估值方法建议
6. 支付安排设计（现金/股份/混合、分期/业绩对赌）
7. 交割条件与过渡期安排
8. 生成交易方案建议书
9. 出稿前审查

## 核心决策框架

### 股权收购 vs 资产收购

| 维度 | 股权收购 | 资产收购 |
|:---|:---|:---|
| 交易标的 | 目标公司股权 | 特定资产/业务 |
| 债务承继 | 承继目标公司全部债务 | 不承继（约定除外） |
| 税负 | 所得税为主 | 增值税+所得税+土增税等 |
| 程序 | 工商变更登记 | 资产过户（多部门） |
| 审批 | 视行业监管要求 | 视资产性质 |
| 适用场景 | 希望获得完整经营实体 | 只想要部分业务/规避或有负债 |

### 交易结构类型

1. **直接收购**：买方直接收购目标股权/资产
2. **SPV收购**：设特殊目的公司收购，隔离风险
3. **增资扩股**：目标公司增发，买方认购
4. **合并**：吸收合并或新设合并

## 税务筹划要点

| 税种 | 股权收购 | 资产收购 |
|:---|:---|:---|
| 企业所得税 | 卖方缴纳（转让所得） | 卖方缴纳 |
| 增值税 | 上市公司股票视情况 | 不动产/无形资产转让 |
| 土地增值税 | 一般无 | 不动产转让（重大影响） |
| 印花税 | 按转让金额万分之五 | 按转让合同贴花 |
| 契税 | 无 | 受让方缴纳（不动产） |
