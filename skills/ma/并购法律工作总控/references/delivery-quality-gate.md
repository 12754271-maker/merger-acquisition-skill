# 交付质量门

## 适用场景

凡并购任务涉及以下正式交付物，必须在生成正文或文件前完成质量门检查：

- 尽职调查报告（含法律意见）
- 交易方案建议书
- 交易文件（SPA/SHA/APA等）
- 交割备忘录
- 法律意见书
- 客户正式交付材料

## 成果类型分类

| 类型 | 说明 | 要求 |
|:---|:---|:---|
| 工作草稿 | 律师内部临时使用 | 文件名标注"草稿" |
| 律师内部报告 | 含策略分析、风险评估 | 必须有读取复查 + 法规校验 |
| 客户正式材料 | 提交客户的法律文件 | 完整质量门检查 |
| 提交监管材料 | 提交市场监管/商务/发改等 | 双人复核建议 |

## 出稿前检查清单

正式交付物须通过以下检查：

- [ ] 文件名和路径正确（非临时缓存目录）
- [ ] 已完成完整读取和读取复查摘要
- [ ] 法规引用已完成核验和时效性检查
- [ ] 关键数据（主体、金额、日期、比例）已二次核对
- [ ] 来源边界已标注
- [ ] 涉及用户决策的事项已列出并等待确认
- [ ] 占位符（甲方/乙方/金额/日期等）已填充或标注待确认
- [ ] 文档结构完整（无空白章节、无断链）

## 拦截规则

正式交付物未通过出稿前检查时：
1. 输出具体未通过项和原因
2. 说明下一步补正路径
3. 不得以"差不多了"、"内容已有"为由跳过
