## 组合场景路由

### 场景一：用户说"帮我做这个收购项目"

1. 先询问交易类型（股权/资产）、目标行业、交易金额大致范围
2. 确认后，按以下顺序建议工作流：
   - 先做尽调方案 → 尽调 → 发现问题
   - 再做交易方案设计 → 确定结构
   - 起草交易文件 → 谈判
   - 准备交割 → 交割 → 整合

### 场景二：用户只上传了一份文件

1. 识别文件类型
2. 如为尽调材料/目标公司资料 → 路由 并购尽职调查
3. 如为合同草稿 → 路由 并购交易文件（审查模式）
4. 如为交易结构备忘录 → 路由 并购交易方案设计

### 场景三：用户描述交易但没说阶段

列出可能阶段，请用户选择：
1. 我要做尽调
2. 我要设计交易方案
3. 我要起草交易文件
4. 我要准备交割
