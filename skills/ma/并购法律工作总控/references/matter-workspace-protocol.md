# 事项工作区协议

## 目录结构说明

每个并购交易项目应建立以下标准目录：

```
【工作区根目录】/
├── _系统记录/
│   ├── 当前事项.md           ← 事项索引，记录当前激活项目
│   ├── 事项列表.md           ← 所有项目列表
│   └── 复盘台账.md           ← 项目复盘记录
├── [客户名-项目代号]/
│   ├── 01-基础信息/          ← 目标公司基本信息、证照
│   ├── 02-尽职调查/          ← 尽调资料、核查记录、DD报告
│   ├── 03-交易方案/          ← 交易结构方案、税务分析
│   ├── 04-交易文件/          ← SPA/SHA/APA及附件
│   ├── 05-交割阶段/          ← 交割清单、交割备忘录
│   ├── 06-整合/              ← 整合方案、整合记录
│   ├── 07-交付文件/          ← 客户正式交付版本
│   └── 08-通信记录/          ← 客户沟通记录、谈判纪要
```

## 环境变量配置

| 变量名 | 说明 | 示例 |
|:---|:---|:---|
| `MA_WORKSPACE` | 工作区根目录 | `/Users/lawyer/MAProjects` |
| `MA_CURRENT_MATTER` | 当前事项路径 | `/Users/lawyer/MAProjects/_系统记录/当前事项.md` |
| `MA_DD_CHECKLIST_DIR` | 尽调清单模板库 | `[自定义路径]` |
| `MA_TEMPLATE_DIR` | 交易文件模板库 | `[自定义路径]` |
| `MA_LAW_DB` | 法规数据库路径或凭证 | `[自定义配置]` |

## 事项切换规则

1. 用户指定客户名或项目代号 → 在事项列表中查找匹配项目
2. 用户提到新项目 → 先建立项目目录结构，更新事项列表和当前事项
3. 用户未指定 → 使用当前激活事项
4. 当前事项不匹配时 → 提示用户确认或切换

## 禁止事项

- 不得在未确认当前事项时写入正式交付文件
- 不得跨项目读取客户材料
- 不得将临时缓存目录作为正式交付路径
